# Estrutura da Landing Page

## Objetivo

A landing page deve apresentar Rafa como um desenvolvedor full-stack que constroi software com metodo, clareza e IA estruturada.

O objetivo nao e apenas listar tecnologias. A Home deve comunicar:

- maturidade tecnica;
- processo de trabalho;
- capacidade de transformar problema em produto;
- uso criterioso de IA com SDD, TDD e context engineering;
- resultados concretos.

## Regra de estrutura

Tudo sera uma unica Home em formato de landing page. As specs e secoes devem ser nomeadas como partes da Home, nao como paginas separadas.

## Ordem das secoes da Home

1. Hero
2. Processo
3. Cases selecionados
4. Stack e principios
5. Sobre
6. Contato

## 1. Hero

Funcao:

- gerar impacto visual imediato;
- apresentar o posicionamento central;
- deixar claro o diferencial do Rafa.

Conteudo base:

```txt
RAFAEL PINHEIRO

Eu uso IA como parte de um processo de engenharia, nao como atalho.

Transformo contexto em especificacao, especificacao em testes, e testes em software que da para confiar.
```

Descricao complementar:

```txt
Desenvolvedor full-stack com foco em TypeScript, React, Next.js e Node.js. Crio aplicacoes web combinando produto, arquitetura, testes e IA estruturada com SDD, TDD e context engineering.
```

Anchors/atributos:

- Full-stack
- SDD/TDD
- Context engineering

CTAs:

- Ver cases
- Conhecer meu processo

Direcao visual:

- paleta bordo como primeira impressao;
- composicao editorial;
- tipografia grande;
- alto contraste;
- possivel elemento lateral forte, sem competir com a leitura.

## 2. Processo

Funcao:

- explicar como a IA entra no processo de engenharia;
- diferenciar metodo de uso superficial de ferramenta;
- transformar a frase central em fluxo compreensivel.

Fluxo base:

```txt
Contexto -> Especificacao -> Testes -> Implementacao -> Validacao
```

Mensagem:

```txt
Antes de pedir codigo para uma IA, eu preparo o terreno: contexto, criterios, restricoes, testes e validacao. Isso reduz ambiguidade e aumenta confianca no resultado.
```

## 3. Cases Selecionados

Funcao:

- provar capacidade tecnica com evidencias;
- mostrar impacto real;
- conectar decisao tecnica, produto e resultado.

Cases iniciais:

- Endpoint otimizado de aproximadamente 50s para 4s.
- Fluxo manual reduzido de aproximadamente 5min para 1min.
- Case de IA/context engineering a definir.

Formato de cada case:

```txt
Problema
Contexto
Papel do Rafa
Decisoes tecnicas
Resultado
Aprendizados
```

Observacao:

Dados internos, clientes e informacoes sensiveis devem ser anonimizados.

## 4. Stack e Principios

Funcao:

- mostrar repertorio tecnico sem virar uma lista enorme de logos;
- agrupar ferramentas por papel no processo.

Grupos:

- Frontend: React, Next.js, Tailwind CSS, Zustand, React Hook Form, Zod.
- Backend: Node.js, Fastify, NestJS, Prisma, PostgreSQL.
- Qualidade: TDD, Vitest, Jest, Clean Code, performance, acessibilidade.
- IA aplicada: SDD, context engineering, revisao assistida, documentacao viva.

Mensagem:

```txt
Tecnologia entra como meio, nao como fim. O foco e escolher ferramentas que deixem o produto mais claro, testavel e facil de evoluir.
```

## 5. Sobre

Funcao:

- apresentar contexto humano e trajetoria;
- reforcar que Rafa atua de ponta a ponta;
- conectar engenharia, produto e aprendizado continuo.

Pontos:

- Desenvolvedor full-stack.
- Programo desde 2021.
- Estudante de Engenharia Mecatronica na UnB.
- Interesse por produto, arquitetura, qualidade e uso estruturado de IA.

## 6. Contato

Funcao:

- fechar com chamada direta;
- facilitar conversa profissional.

CTAs possiveis:

- Entrar em contato
- Ver GitHub
- Ver LinkedIn

Mensagem:

```txt
Se voce quer construir software com clareza, metodo e cuidado tecnico, vamos conversar.
```
